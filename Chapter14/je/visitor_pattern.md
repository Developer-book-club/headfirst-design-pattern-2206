# Visitor Pattern

> 알고리즘을 객체 구조에서 분리시키는 디자인 패턴
>
> 방문자와 방문 공간을 분리하여 방문 공간이 방문자를 맞이할 때, 이에 대한 행동을 방문자에게 위임하는 패턴

다양한 객체에 새로운 기능을 추가해야 하는데 캡슐화가 별로 중요하지 않다면 비지터 패턴을 사용한다.

트래버서(Traverser)객체와 함께 사용해서 각각의 상태를 들고옴 <br>→ 새로운 기능이 필요하면 비지터만 변경하면 됨.
<br>(트래버서는 컴포지트 패턴에서 복합 객체 내의 모든 객체에 접근하는 일을 도와줌)

- 장점
  - 구조를 변경하지 않으면서도 복합 객체 구조에 새로운 기능 추가 가능
  - 비교적 손쉽게 새로운 기능 추가 가능
  - 비지터가 수행하는 기능과 관련된 코드를 한곳에 모아 둘 수 있다.
- 단점
  - 비지터를 사용하면 복합 클래스의 캡슐화가 깨진다.
  - 컬렉션 내의 모든 항목에 접근하는 트래버서가 있으므로 복합 구조를 변경하기가 더 어려워진다.
